# 회의 기록 시간: 10:00-12:00

## 프론트엔드 및 UI/UX

- 디자인 테마
- **메인 페이지:**
    - 고객 유형 및 특성 정보 노출.
    - 메인 배너 헤더에 캐릭터를 활용한 분석 결과 배치.
    - 미완성 부분은 시스템 내부 링크로 연결 처리.
- **상품 및 요금제:**
    - 상품 페이지에서 요금 페이지 URL로 연결.
    - 요금제 변경 시, 추천 요금제와의 가격 차이(예: -10,000원) 명시.
    - 부가서비스 개수(+2)를 숫자로 시각화하여 노출.
- **공지사항:** 일반 페이지가 아닌 **팝업 모달** 형식으로 구현.
- **추천 UI:** 클릭해서 이동하는 방식이 아니라, 메인 화면에 **직접 컨텐츠를 띄우는 방식** 채택.

### 주요 기능 명세 및 로직

- **요금제 변경 및 패턴 분석:**
    - 실시간 데이터 사용 패턴 분석 반영.
    - 고객 유형이 하루 단위로 바뀔 수 있으므로 **배치(Batch) 작업**으로 처리.
- **고객 유형 분류 (CRM 기준):**
    - **고객 시점:** 5\~6가지 유형으로 분류.
    - **관리자 시점:** 이탈 잠재 유형 등 CRM 데이터 기반 관리.
- **추천 시스템:**
    - 특정 유형군(Cluster)에 맞는 상품 추천 ("\~것을 더 선호했어요!" 문구 활용).
    - 사용자 필터링 기반의 추천 로직 적용.

### 백엔드 및 인프라

- **메시지 큐(Message Queue):** 코어 API 변경에 따른 메시지 큐 연동.
- **이벤트 처리:** 실제 이벤트 발생 시 외부 링크 연동.
- **데이터 활용:** 고객 데이터 기반 추천 로직 고도화.

### 관리자 및 상담사 페이지

- **관리자용 기능:** 지라(Jira) 에픽 생성 및 관리자 전용 CRM 뷰.
- **상담사 페이지:** 고객 분석 데이터를 바탕으로 한 상담 지원 화면 구성.

## 🚀 향후 과제 (Next Steps)

1. **배치 프로세스 설계:** 하루 단위로 갱신되는 고객 유형 업데이트 로직 구체화.
2. **UI 컴포넌트 개발:** 보라/네이비 톤의 요금제 비교 카드 및 팝업 모달 디자인.
3. **API 명세 업데이트:** 메시지 큐 도입에 따른 데이터 흐름도 정리.
