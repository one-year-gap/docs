### 관리자 - 페이지네이션

- 페이지 번호(page) + 페이지 리미트(totalCount/pagesize) + 한 페이지에서 몇 명의 데이터를 받아올건지(pagesize)를 파라미터로 전송

      "id":"0001",
      "grade":"VIP",

      "name":"고객1",

      "phone":"010-\*\*\*\*-1234",

### 관리자 - 고객 상세 정보 조회

- 어떤 정보를 요청할지 → 우선은 피그마 기준으로 퍼블리싱 작업 진행
- 캐릭터 유형 페이지의 존재 타당성 강화
    - 마이페이지와 차별화 필요 → 차별화가 되어야 캐릭터 유형 페이지의 존재 이유가 명확해짐
    - 백엔드분 피드백 → 서비스 시작 시 화살표로 가리키기 (페이지 이동 유도)

### 고객 - 메인 페이지

- 메인 페이지 구성 고민중

### 고객 - 캐릭터 추천 로직

- 회원가입 시 캐릭터 유형이 바로 추천되는지?
→ 제안
    1. 회원가입 시 간단한 설문조사 추가
    2. 고객 기본 정보로 캐릭터 유형 분류
    3. 기본 캐릭터로 두고, 이용 정보 쌓였을 때 추천해주기 (화살표로 표시)

### 고객 - 로그인

- 구글 소셜 로그인 시
    - 이름, 이메일만 줌
    - 따라서 이름, 이메일이 채워진 상태로 회원가입 페이지로 이어짐
- 구글 로그인 하다가 나갔다가 들어왔을때 정보가 남아있어야함
